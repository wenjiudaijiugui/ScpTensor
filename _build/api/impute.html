Imputation¶

scptensor.impute.knn(container, assay_name, base_layer, new_layer_name='imputed_knn', k=5, weights='uniform', batch_size=500, oversample_factor=3)[source]¶

Impute missing values using k-Nearest Neighbors with Over-sampling and Filtering.

Refactored V4: 1. Time Complexity: Dominated by distance calculation O(M * N * D). 2. Space Complexity: O(Batch * N) due to batch processing. 3. Algorithm Correctness: Addresses “Effective K Decay” by over-sampling neighbors.

We search for k_search = oversample_factor * k neighbors first. Then, for each missing feature, we filter these candidates to find valid (non-missing) ones. We select the top k valid neighbors for imputation.

  1. Numerical Stability: Adds protection for small weights sum in weighted KNN.

Args:

container: The ScpContainer object. assay_name: Name of the assay to use. base_layer: Name of the layer containing data with missing values. new_layer_name: Name for the new layer with imputed data. k: Number of neighbors to use for calculation. weights: Weight function used in prediction. ‘uniform’ or ‘distance’. batch_size: Number of rows to process at once to control memory usage. oversample_factor: Factor to determine search range (k_search = oversample_factor * k).

Higher values reduce the chance of effective k decay but increase computation slightly.

Returns:

ScpContainer: The updated container with the new layer.

Raises:

AssayNotFoundError: If the specified assay does not exist. LayerNotFoundError: If the specified layer does not exist in the assay. ScpValueError: If k, batch_size, weights, or oversample_factor parameters are invalid.

Impute missing values using MissForest (Random Forest Imputation).

This implementation follows the MissForest algorithm: 1. Initial imputation using mean/median. 2. Iteratively impute each variable (feature) by training a Random Forest

on observed values of other variables.

  1. Stop when the difference between iterations is small or max_iter is reached.

Args:

container: The ScpContainer object. assay_name: Name of the assay to use. base_layer: Name of the layer containing data with missing values. new_layer_name: Name for the new layer with imputed data. max_iter: Maximum number of iterations. n_estimators: Number of trees in the forest. max_depth: Maximum depth of the trees. n_jobs: Number of jobs to run in parallel. random_state: Random seed for reproducibility. verbose: Verbosity level.

Returns:

ScpContainer: The updated container with the new layer.

Raises:

AssayNotFoundError: If the specified assay does not exist. LayerNotFoundError: If the specified layer does not exist in the assay. ScpValueError: If max_iter, n_estimators, or other parameters are invalid.

scptensor.impute.ppca(container, assay_name, base_layer, new_layer_name='imputed_ppca', n_components=10, max_iter=100, tol=1e-06, random_state=None)[source]¶

Impute missing values using Probabilistic PCA (PPCA).

PPCA models the data as a linear Gaussian latent variable model and uses the EM algorithm to estimate parameters and impute missing values. This method is effective for data with a strong low-dimensional structure.

Parameters:

  • container (ScpContainer) – Input container with missing values

  • assay_name (str) – Name of the assay to use

  • base_layer (str) – Name of the layer containing data with missing values

  • new_layer_name (str, optional) – Name for the new layer with imputed data (default: ‘imputed_ppca’)

  • n_components (int, optional) – Number of principal components to use (default: 10)

  • max_iter (int, optional) – Maximum number of EM iterations (default: 100)

  • tol (float, optional) – Convergence tolerance (default: 1e-6)

  • random_state (int, optional) – Random seed for reproducibility (default: None)

Returns:

Container with imputed data in new layer

Return type:

ScpContainer

Raises:

  • AssayNotFoundError – If the specified assay does not exist

  • LayerNotFoundError – If the specified layer does not exist in the assay

  • ScpValueError – If n_components, max_iter, or tol parameters are invalid

  • DimensionError – If n_components is too large for the data dimensions

Notes

The PPCA model assumes:

x = Wz + mu + epsilon

where:

  • z is a latent variable with N(0, I)

  • W is the factor loading matrix

  • mu is the mean

  • epsilon is noise with N(0, sigma^2 I)

The EM algorithm iteratively:

  1. E-step: Compute posterior expectations of latent variables

  2. M-step: Update W and sigma^2

  3. Impute missing values using current model parameters

Examples

>>> container = ppca(container, assay_name='protein', base_layer='raw')
>>> # Impute using 20 principal components
>>> container = ppca(container, assay_name='protein',
...                  base_layer='raw', n_components=20)
scptensor.impute.svd_impute(container, assay_name, base_layer, new_layer_name='imputed_svd', n_components=10, max_iter=100, tol=1e-06, init_method='mean')[source]¶

Impute missing values using iterative SVD.

This method alternates between:

  1. Computing SVD of the current (partially filled) matrix

  2. Reconstructing the matrix using only top singular values/vectors

  3. Filling missing values with the reconstructed values

Parameters:

  • container (ScpContainer) – Input container with missing values

  • assay_name (str) – Name of the assay to use

  • base_layer (str) – Name of the layer containing data with missing values

  • new_layer_name (str, optional) – Name for the new layer with imputed data (default: ‘imputed_svd’)

  • n_components (int, optional) – Number of singular components to use (default: 10)

  • max_iter (int, optional) – Maximum number of iterations (default: 100)

  • tol (float, optional) – Convergence tolerance (default: 1e-6)

  • init_method (str, optional) – Initialization method: ‘mean’ or ‘median’ (default: ‘mean’)

Returns:

Container with imputed data in new layer

Return type:

ScpContainer

Raises:

  • AssayNotFoundError – If the specified assay does not exist

  • LayerNotFoundError – If the specified layer does not exist in the assay

  • ScpValueError – If n_components, max_iter, tol, or init_method parameters are invalid

  • DimensionError – If n_components is too large for the data dimensions

Notes

The algorithm follows these steps:

  1. Initialize missing values (mean or median)

  2. Repeat until convergence: a. Compute SVD: X = U @ S @ V.T b. Keep only top k components: X_k = U[:, :k] @ S[:k] @ V[:, :k].T c. Fill missing values with X_k d. Check convergence (change in imputed values)

This is equivalent to soft-impute with nuclear norm regularization and is effective for low-rank data structures.

Examples

>>> container = svd_impute(container, assay_name='protein', base_layer='raw')
>>> # Impute using 20 singular components
>>> container = svd_impute(container, assay_name='protein',
...                        base_layer='raw', n_components=20)

This module implements PPCA-based missing value imputation using an expectation-maximization (EM) algorithm.

Reference:

Tipping ME, Bishop CM. Probabilistic principal component analysis. Journal of the Royal Statistical Society (1999).

This module implements SVD-based missing value imputation, which iteratively fills missing values using low-rank matrix approximation.

Troyanskaya O, et al. Missing value estimation methods for DNA microarrays. Bioinformatics (2001).
