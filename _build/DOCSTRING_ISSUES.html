1. Duplicate Object Descriptions (22 warnings)¶

2. Docstring Formatting Issues (40+ warnings)¶

These are formatting issues in docstrings that need to be fixed:

Definition List / Indentation Issues¶

File: scptensor/core/structures.py

  • ScpContainer.__init__: Line 4 - Definition list needs blank line

  • Assay.__init__: Line 4 - Definition list needs blank line

File: scptensor/dim_reduction/pca.py

  • pca function: Line 13 - Block quote ends without blank line

File: scptensor/impute/knn.py

  • knn function: Line 10 - Block quote ends without blank line

File: scptensor/impute/missforest.py

  • missforest function: Line 7 - Block quote ends without blank line

And similar issues in other modules.

Solution: Add blank lines between parameter descriptions and example sections in docstrings.

Example fix:

# Before (causes warning):
def foo(x, y):
    """
    Function description.

    Parameters
    ----------
    x : int
        First param
    y : int
        Second param
    Examples
    --------
    >>> foo(1, 2)
    """

# After (fixed):
def foo(x, y):
    """
    Function description.

    Parameters
    ----------
    x : int
        First param
    y : int
        Second param

    Examples
    --------
    >>> foo(1, 2)
    """
