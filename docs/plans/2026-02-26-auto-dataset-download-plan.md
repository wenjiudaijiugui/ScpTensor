# 自动化数据集下载系统 - 实现计划

## 概述

本文档详细说明自动化数据集下载系统的实现计划，包括任务分解、依赖关系和预估工作量。

## 任务列表

### Task 1: 创建 auto_download.py 主脚本框架

**文件**: `scptensor/datasets/scripts/auto_download.py`

**内容**:
- 命令行参数解析 (argparse)
- 主流程框架 (main function)
- 日志配置 (logging)
- 入口点

**预估代码**: ~80 行

**验收标准**:
- 支持 --max, --threshold, --output, --report, --dry-run 参数
- 日志输出到控制台和文件

---

### Task 2: 实现候选搜索模块

**功能**: 从 PRIDE API 获取候选数据集列表

**实现**:
- 复用 `search_datasets.py` 的 PRIDE API 调用逻辑
- 搜索关键词: "single cell proteomics"
- 返回候选数据集列表 (包含 PRIDE ID、标题、基本信息)

**预估代码**: ~60 行 (复用现有代码)

**验收标准**:
- 成功获取候选数据集列表
- 处理 API 分页

---

### Task 3: 实现网络审查模块

**功能**: 对每个候选数据集进行详细审查

**实现**:
- 访问 PRIDE 项目页面获取详情
- 检查文件列表 (识别报告文件类型)
- 验证仪器类型 (从元数据提取)
- 提取发表信息 (DOI, PubMed ID)
- 估算样本规模

**预估代码**: ~150 行

**验收标准**:
- 正确解析 PRIDE 项目页面
- 提取评分所需的各项信息
- 处理网络错误和超时

---

### Task 4: 实现评分系统

**功能**: 根据评分标准计算每个数据集的总分

**评分维度**:
1. **数据完整性 (40%)**
   - 有报告文件: 30 分
   - 格式正确 (report.tsv): 10 分

2. **仪器类型 (30%)**
   - Orbitrap Astral: 30 分
   - Orbitrap Exploris: 20 分
   - 其他: 0 分

3. **发表质量 (20%)**
   - DOI + PubMed ID: 20 分
   - 仅 DOI: 10 分
   - 无发表信息: 0 分

4. **样本规模 (10%)**
   - >= 100 样本: 10 分
   - >= 50 样本: 5 分
   - < 50 样本: 2 分

**预估代码**: ~80 行

**验收标准**:
- 评分结果在 0-100 范围内
- 各维度评分可独立测试

---

### Task 5: 实现自动下载模块

**功能**: 下载评分达标的数据集

**实现**:
- 复用 `download_dataset.py` 的下载逻辑
- 支持断点续传 (检查已下载文件)
- 错误处理和重试机制
- 下载进度显示

**预估代码**: ~80 行 (复用现有代码)

**验收标准**:
- 正确下载所有必需文件
- 支持断点续传
- 下载失败时记录错误

---

### Task 6: 实现 Markdown 报告生成

**功能**: 生成详细的审查报告

**报告内容**:
1. **摘要统计**
   - 候选数据集总数
   - 已下载数量
   - 跳过数量
   - 待审核数量

2. **已下载数据集详情**
   - PRIDE ID
   - 标题
   - 评分明细
   - 文件列表

3. **跳过的数据集**
   - PRIDE ID
   - 跳过原因
   - 评分

4. **待人工审核列表**
   - 接近阈值的数据集 (50-59 分)
   - 信息不完整的数据集

**预估代码**: ~100 行

**验收标准**:
- 报告格式正确
- 信息完整准确

---

### Task 7: 测试和验证

**测试项目**:
1. **dry-run 模式测试**
   - 验证搜索和评分逻辑
   - 不实际下载文件

2. **完整流程测试**
   - 使用小型测试数据集
   - 验证端到端流程

3. **报告格式验证**
   - 检查 Markdown 格式正确性
   - 验证链接有效性

**预估时间**: 2-3 小时

---

## 依赖

- **requests**: HTTP 请求 (已有)
- **beautifulsoup4**: HTML 解析 (如需要，需添加)
- **现有脚本模块**: 复用搜索、下载逻辑

## 命令行接口

```bash
# 基本用法 - 审查并下载
uv run python scptensor/datasets/scripts/auto_download.py

# 仅审查不下载
uv run python scptensor/datasets/scripts/auto_download.py --dry-run

# 自定义参数
uv run python scptensor/datasets/scripts/auto_download.py \
    --max 10 \
    --threshold 70 \
    --output ./downloaded_datasets \
    --report auto_download_report.md
```

## 风险和缓解

| 风险 | 缓解措施 |
|------|----------|
| PRIDE API 变更 | 使用稳定的 API 端点，添加版本检查 |
| 网络不稳定 | 实现重试机制和超时处理 |
| 评分标准不适用 | 设计可配置的评分权重 |
| 存储空间不足 | 提前检查可用空间 |
