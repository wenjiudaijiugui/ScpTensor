# 自动化数据集下载系统设计

**日期**: 2026-02-26
**状态**: 已批准

## 概述

创建自动化系统，从 PRIDE 数据库搜索、审查并下载高质量单细胞蛋白质组学数据集。

## 需求

- **触发方式**: 一次性批量下载
- **审查方式**: 调用网络搜索工具获取详细信息
- **判断优先级**: 数据完整性 > 仪器类型 > 发表质量 > 样本规模
- **输出**: Markdown 详细报告

## 架构

```
auto_download.py
├── 1. 搜索候选 (PRIDE API)
├── 2. 网络审查每个候选
├── 3. 评分排序
├── 4. 自动下载高分数据集
└── 5. 生成 Markdown 报告
```

## 评分标准

| 维度 | 权重 | 评分细则 |
|------|------|----------|
| 数据完整性 | 40% | 有报告文件(30) + 格式正确(10) |
| 仪器类型 | 30% | Astral(30) / Exploris(20) / 其他(0) |
| 发表质量 | 20% | DOI+PubMed(20) / 仅DOI(10) / 无(0) |
| 样本规模 | 10% | >=100(10) / >=50(5) / <50(2) |

**下载阈值**: 总分 >= 60 分

## 命令行接口

```bash
uv run python scptensor/datasets/scripts/auto_download.py
--max N          # 最多下载 N 个数据集
--threshold N    # 评分阈值 (默认 60)
--output DIR     # 输出目录
--report FILE    # 报告文件名
--dry-run        # 仅审查不下载
```

## 报告格式

Markdown 格式，包含:
- 摘要统计
- 已下载数据集详情
- 跳过的数据集及原因
- 待人工审核列表

## 错误处理

- 网络请求超时: 重试 3 次，间隔 5 秒
- 下载中断: 支持断点续传
- 评分信息缺失: 使用默认值 0
- 存储空间不足: 提前检查并警告

## 扩展性

系统设计支持未来扩展:

1. **新数据源**: 可添加其他数据库 (ProteomeXchange, MassIVE)
2. **新评分维度**: 模块化评分系统便于添加新维度
3. **定时任务**: 可配置为定期检查新数据集
4. **通知机制**: 可集成邮件/Slack 通知下载完成
